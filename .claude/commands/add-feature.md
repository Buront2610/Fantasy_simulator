機能: $ARGUMENTS

以下の手順で実装してください:

1. **調査**: 関連する既存コードを読んで、アーキテクチャと実装パターンを理解する
2. **計画**: 変更が必要なファイルと影響範囲を特定し、実装計画を提示する
3. **ユーザー確認**: 計画について確認を取る
4. **実装**:
   - 既存のコーディング規約に従う
   - ユーザー向け文字列はすべて `t()` 経由（i18n）
   - シリアライゼーションが必要な場合は `to_dict()` / `from_dict()` パターン
5. **テスト**: 対応するテストファイルにテストを追加
6. **検証**: Lint + テストを一括実行
7. **報告**: 変更内容のサマリーを報告

ルール:
- 外部パッケージを追加しない
- 既存のテストを壊さない
- インクリメンタルに変更する
- 変更は最小限に、シンプルに保つ
