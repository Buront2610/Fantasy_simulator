i18n.pyに新しい翻訳文字列を追加してください。

対象: $ARGUMENTS

以下の手順で実装してください:

1. `i18n.py` を読んで既存の翻訳キー構造と命名規則を確認する
2. 指定された対象（ファイルやモジュール）を読んで、ハードコードされた文字列を探す
3. 翻訳キーを適切な階層で命名する（例: `"screens.menu.title"`, `"events.battle.victory"`）
4. `i18n.py` の `_TEXT` 辞書に英語と日本語の両方を追加する（locale-first構造: `_TEXT["ja"]["key"]`, `_TEXT["en"]["key"]`）。用語は `_TERMS` に追加
5. 元のコードで `tr("key", **kwargs)` を使うように変更する（`tr()` は内部でformatするので `.format()` 連鎖は不要）。用語は `tr_term("term")`
6. Lintとテストを実行して確認する

ルール:
- 既存のキー命名規則に従う
- 英語と日本語の両方を必ず追加する
- フォーマット文字列が必要な場合は `{variable}` プレースホルダーを使い、`tr("key", variable="value")` で呼ぶ
